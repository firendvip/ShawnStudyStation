# 测试用例：全站设置图标统一为 PinPin 齿轮形状

## 1. 概述

### 1.1 改动背景
将全站各应用页面的「设置」入口图标统一为 **PinPin 错字练习** 所使用的同款齿轮图标（Feather 风格齿轮），消除各页图标不一致（如 Font Awesome `fa-cog`、滑块图标等）的问题，形成统一的视觉语言。

### 1.2 基准定义（Baseline）
基准图标来源：PinPin（错字练习）`vendor/PinPin/app/page.tsx` 中的设置齿轮 SVG，其确切结构为：

```html
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor"
     stroke-width="1.8" stroke-linecap="round" stroke-linejoin="round" aria-hidden="true">
  <circle cx="12" cy="12" r="3" />
  <path d="M19.4 15a1.65 1.65 0 0 0 .33 1.82l.06.06a2 2 0 1 1-2.83 2.83l-.06-.06a1.65 1.65 0 0 0-1.82-.33 ...z" />
</svg>
```

关键特征（判定「同款齿轮」的依据）：
- `viewBox="0 0 24 24"`
- 内圆 `<circle cx="12" cy="12" r="3">`
- 齿轮外轮廓为 Feather 风格单一 `<path>`（以 `M19.4 15` 开头）
- `fill="none"` + `stroke="currentColor"`（颜色随上下文继承，不写死色值）
- `stroke-linecap/linejoin="round"`

### 1.3 改动范围（受测页面与真实选择器）
经代码核对，各页设置图标现状如下：

| 页面 | 目录 | 设置入口选择器 | 当前图标 | 打开的面板 |
|------|------|----------------|----------|-----------|
| PinPin 错字练习（基准） | `vendor/PinPin/` | 设置齿轮（page.tsx） | Feather 齿轮 | 设置面板 |
| 单词小测 wordquiz | `vendor/wordquiz/index.html` | `#settings-btn`（`.settings-icon`） | 内联 Feather 齿轮 SVG（已由 fa-cog 改为内联） | 设置弹窗 |
| 小学心算 koudaa | `vendor/koudaa/index.html` | `#help-btn`（内含 `svg.settings-icon`） | 内联齿轮 SVG（原滑块已改），白色齿轮/半透明白底圆钮，位于绿色渐变背景上 | `#instruction-modal`（含语速滑块 / 使用说明） |
| 转盘 lottery | `vendor/lottery/index.html` | `#settings-btn`（`.icon-btn`） | 内联 Feather 齿轮 SVG（`stroke-width="2"`） | 设置面板 |
| 单词速刷 CoCaWordCards | `vendor/CoCaWordCards/webapp/index.html` | `#settingsFab`（`.settings-fab` 常驻浮动按钮） | 内联 Feather 齿轮 SVG | 设置面板 |

> 说明：wordquiz/lottery/CoCaWordCards 齿轮 `stroke-width` 存在 1.8 与 2 的细微差异，属可接受范围（不影响「同款齿轮」形状判定）。本套用例以「同一齿轮 path + 内圆 r=3 + viewBox 24」作为一致性判定标准。

### 1.4 与需求描述不符 / 需注意之处
- 需求把 koudaa 设置入口称为 `#settings-btn`，**实际选择器为 `#help-btn`**（其内层 SVG 类名为 `settings-icon`），点击后打开 `#instruction-modal`。测试脚本请以 `#help-btn` 为准。
- CoCaWordCards 设置入口为 `#settingsFab`（常驻浮动按钮），**非** `#settings-btn`。
- koudaa 的「绿底白色」实为：图标 `color: white`（`currentColor` 继承为白），按钮 `.help-btn` 为半透明白底圆钮，叠加在页面绿色渐变背景之上；并非「纯绿底填充齿轮」。
- 主站 `index.html` 内**未发现** `look3`、`cb=`（cache-buster）等硬编码字样；`apps/` 与 `iframe` 引用存在（主站通过 iframe 聚合子应用）。云端 look3.cn 与独立页 `/apps/...` 的跨环境用例按需求描述编写，执行时以线上实际路径为准。
- 经核对，wordquiz 已无 `fa-cog`、koudaa 已由滑块改为齿轮、lottery/CoCaWordCards 已是同款齿轮——四页均已改动到位，未发现「某页尚未改」的情况。

## 2. 测试环境与前置说明

### 2.1 环境
- **本地**：直接以本地文件 / 本地静态服务器打开各 `vendor/<app>/index.html`。
- **云端**：`look3.cn` 主站（通过 iframe 聚合）及独立页 `/apps/<app>/`。
- **浏览器**：Chrome / Firefox / Safari 最新稳定版（至少 Chrome + 一款其它内核）。

### 2.2 通用前置条件
- 已获取最新代码 / 最新线上部署。
- 云端验证时对目标 URL 追加 cache-buster 参数（如 `?cb=<时间戳>`）或强制刷新（Cmd/Ctrl+Shift+R），确保加载到最新静态资源。
- 打开浏览器开发者工具 Console，用于观察 JS 报错。

### 2.3 判定术语
- **同款齿轮**：SVG 含 `viewBox="0 0 24 24"`、`<circle ... r="3">`、以 `M19.4 15` 开头的 Feather 齿轮 `<path>`、`fill="none"`、`stroke="currentColor"`。
- **正常打开设置面板**：点击设置入口后对应弹窗/面板可见且可交互。

## 6. 执行建议

1. 先执行 P0（形状一致性 + 各页点击回归 + 跨环境 + 无报错），构成冒烟。
2. 断网/断 FA 类边界用例（TC-WQ-002、TC-ERR-003）建议用浏览器网络面板屏蔽资源模拟。
3. 云端用例务必带 cache-buster 或强刷，避免旧缓存造成误判。
4. 形状一致性优先以源码 path 比对（TC-SHAPE-007）作为客观判定，截图对比为辅助。
