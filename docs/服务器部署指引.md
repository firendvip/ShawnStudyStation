# 服务器部署指引(小善学习站)

本文档提供一段「可直接复制粘贴」的指令,交给 **Claude Chrome 扩展(浏览器内 Claude)**,
让它驱动 **腾讯云网页终端(OrcaTerm)**,在你的云服务器(CVM)上把项目部署上线。

---

## 给你(本机用户)的准备清单 —— 粘贴前请先备齐

在把下面的「粘贴块」交给浏览器内的 Claude 之前,请确认你已经准备好以下内容:

1. **代码仓库已就绪**:项目已推送到 GitHub 或 Gitee,并拿到 **clone URL**
   (例如 `https://github.com/yourname/ShawnStudyStation.git`)。
2. **域名已备案且解析生效**:你拥有一个 **已完成 ICP 备案** 的域名,并已添加 **A 记录** 指向
   CVM 的 **公网 IP**(国内服务器走 80/443 的网站必须备案)。
3. **安全组已放行**:在腾讯云控制台为该 CVM 的安全组放行 **TCP 80** 和 **TCP 443** 入站。
4. **邮箱验证码 SMTP 配置** 已拿到(来自 `docs/邮箱验证码配置指引.md`):
   - `SMTP_HOST`(如 `smtp.qq.com`)
   - `SMTP_PORT`(默认 `465`)
   - `SMTP_SECURE`(默认 `true`)
   - `SMTP_USER`(发信邮箱)
   - `SMTP_PASS`(邮箱授权码 / 应用专用密码,**不是登录密码**)
   - `SMTP_FROM`(发件人,需与 `SMTP_USER` 同邮箱,如 `小善学习站 <你的邮箱@qq.com>`)
5. **SSL 证书文件** 已准备好:`fullchain.pem`(证书全链)与 `privkey.pem`(私钥)。
   可在 **腾讯云 SSL 证书控制台** 申请并下载 **Nginx 格式** 的证书包后取出这两个文件。

> ⚠️ 重要提醒
> - **私有仓库** 需要在 CVM 上配置 **部署密钥(SSH Deploy Key)** 或使用 **HTTPS 访问令牌(token)** 才能 clone。
> - **密钥与证书绝不要提交进 git**:`.env`、`deploy/certs/*.pem` 已被 `.gitignore` 忽略,请保持如此。

---

## 复制下面整段交给「浏览器内的 Claude 扩展」

> 复制从下一行四个反引号开始、到对应四个反引号结束的全部内容。

````text
你好,Claude。请你充当我的部署向导,**全程通过腾讯云网页终端(OrcaTerm)** 帮我把
「小善学习站」项目部署到我的云服务器(CVM)上。请一步一步带我操作,每一步等我确认后再继续。

【开始前,请先向我索取以下信息】(缺一不可,缺了就先问我):
- 代码仓库 clone URL(GitHub 或 Gitee);若为私有仓库,告诉我用部署密钥还是 HTTPS token。
- 已备案的域名(例如 study.example.com)。
- 邮箱验证码 SMTP 配置:SMTP_HOST、SMTP_PORT、SMTP_SECURE、SMTP_USER、
  SMTP_PASS(授权码 / 应用专用密码)、SMTP_FROM(需与 SMTP_USER 同邮箱)。
- SSL 证书内容:fullchain.pem 与 privkey.pem(可现场粘贴文本,或说明从腾讯云 SSL 控制台下载)。
- 数据库密码与 JWT 密钥:可现场用 `openssl rand -hex 32` 生成,无需我提前准备。

【部署步骤】

1. 打开 CVM 实例列表 https://console.cloud.tencent.com/cvm/instance ,
   选中目标实例,点右侧「登录」,选择 **OrcaTerm 网页终端** 方式登录
   (使用 root,或可 sudo 的用户;后续命令如非 root 请加 sudo)。

2. 安装 Docker Engine 与 compose 插件(Ubuntu / Debian):
   方式 A(官方一键脚本,最简单):
   ```
   curl -fsSL https://get.docker.com | sh
   sudo apt-get update
   sudo apt-get install -y docker-compose-plugin
   ```
   方式 B(官方 apt 仓库,手动步骤):
   ```
   sudo apt-get update
   sudo apt-get install -y ca-certificates curl gnupg
   sudo install -m 0755 -d /etc/apt/keyrings
   curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
   sudo chmod a+r /etc/apt/keyrings/docker.gpg
   echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
   sudo apt-get update
   sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
   ```
   验证安装成功:
   ```
   docker --version
   docker compose version
   ```

3. 安装 git 并克隆仓库:
   ```
   sudo apt-get install -y git
   git clone <仓库URL>
   ```
   说明:`<仓库URL>` 请替换为我提供的真实地址。
   - **私有仓库 + 部署密钥**:先 `ssh-keygen -t ed25519 -C "deploy"` 生成密钥,
     把 `~/.ssh/id_ed25519.pub` 内容添加到仓库的 Deploy Keys,再用 SSH 形式 clone。
   - **私有仓库 + HTTPS token**:用 `git clone https://<token>@github.com/用户名/仓库.git`。

4. 进入部署目录并准备环境变量:
   ```
   cd <项目目录>/deploy
   cp .env.example .env
   nano .env
   ```
   在 `.env` 中填写(其中密码/密钥可现场生成):
   - `DOMAIN=` 我的真实域名(不带 https://,不带斜杠)
   - `POSTGRES_PASSWORD=` 强随机密码,执行 `openssl rand -hex 32` 生成后粘贴
   - `JWT_SECRET=` 强随机密钥,执行 `openssl rand -hex 32` 生成后粘贴
   - `SMTP_*` 邮箱验证码值(用我提供的真实值填入):`SMTP_HOST` / `SMTP_PORT`(默认 465)/
     `SMTP_SECURE`(默认 true)/ `SMTP_USER` / `SMTP_PASS`(授权码 / 应用专用密码)/ `SMTP_FROM`
   - `EMAIL_DEV_MODE=false`(生产必须为 false,走真实邮箱验证码)
   nano 保存:Ctrl+O 回车,Ctrl+X 退出。

5. 放置 SSL 证书到 `deploy/certs/`:
   需要两个文件:`fullchain.pem`(证书全链)与 `privkey.pem`(私钥)。
   ```
   nano certs/fullchain.pem   # 粘贴证书全链内容,Ctrl+O 保存,Ctrl+X 退出
   nano certs/privkey.pem     # 粘贴私钥内容,Ctrl+O 保存,Ctrl+X 退出
   ```
   或者:从 **腾讯云 SSL 证书控制台** 下载 **Nginx 格式** 证书包,
   将其中的 `*.pem`(全链)与 `*.key`(私钥)分别另存为 `fullchain.pem` 与 `privkey.pem`,
   通过 OrcaTerm 的「上传文件」功能上传到 `deploy/certs/` 目录。

6. 编辑 Nginx 配置,把占位的 server_name 改成真实域名:
   ```
   nano nginx/conf.d/default.conf
   ```
   将文件中的 `server_name _;`(共两处:80 跳转块与 443 块)改为:
   ```
   server_name 我的真实域名;
   ```
   保存退出。

7. 上线前确认两件事:
   - 腾讯云该 CVM 的 **安全组已放行 TCP 80 与 443** 入站。
   - 域名的 **A 记录已指向本机公网 IP 且解析已生效**
     (可在终端用 `ping 我的域名` 或 `dig 我的域名` 查看返回的是否为本机公网 IP)。

8. 构建并启动服务(在 `deploy` 目录下):
   ```
   docker compose up -d --build
   ```
   首次构建会拉取镜像并编译,耗时较久,请耐心等待。完成后查看状态:
   ```
   docker compose ps
   ```
   应看到 **四个服务** 都处于运行(Up / healthy)状态:
   `postgres`、`backend`、`pinpin`、`web`。

9. 验证部署:
   ```
   curl https://<域名>/api/health
   ```
   预期返回:`{"ok":true}`。
   然后在浏览器打开 `https://<域名>`,依次测试核心功能:
   - 注册(会收到 **真实邮箱** 验证码)
   - 登录
   - 错字练习
   - 自然拼读

10. 排错(如某步失败,请按服务查看日志):
    ```
    docker compose logs -f backend
    docker compose logs -f pinpin
    docker compose logs -f web
    ```
    常见问题排查方向:
    - **证书路径**:`deploy/certs/` 下是否存在 `fullchain.pem` 与 `privkey.pem`,文件名是否完全一致。
    - **server_name**:`nginx/conf.d/default.conf` 中两处是否都已改成真实域名。
    - **安全组**:80/443 入站是否真的放行;云防火墙是否也需放行。
    - **邮箱凭证**:`SMTP_*` 是否填写正确(`SMTP_PASS` 用的是授权码 / 应用专用密码而非登录密码),`EMAIL_DEV_MODE` 是否为 false,`SMTP_FROM` 是否与 `SMTP_USER` 同邮箱,邮箱是否已开启 SMTP 服务且未触发每日发信上限。

【安全提醒】
- `.env` 与 `certs/*.pem` **绝不要提交进 git**(项目 `.gitignore` 已忽略,请勿强行添加)。
- 定期用 `docker compose exec postgres pg_dump -U <用户> <库名> > backup.sql` 备份数据库。
- 邮箱验证码使用普通 SMTP 发信;个人邮箱有 **每日发信上限**,用户量大时建议改用事务邮件服务(见 `docs/邮箱验证码配置指引.md`)。授权码等同密码,切勿泄露。

请从第 1 步开始,先向我索取上面列出的必要信息,确认齐全后再带我逐步操作。
````
