# 邮箱验证码配置指引（个人开发者也能用,无需企业资质）

本项目用**邮箱验证码**完成注册 / 登录:后端通过通用 SMTP(nodemailer)发送验证码邮件。
个人 / 自用开发者无需任何工商资质,只要有一个支持 SMTP 的邮箱即可。

> 为什么不用短信?个人认证用户已无法使用腾讯云短信(运营商自 2025-09-18 起不支持个人自用签名)。
> 旧的《腾讯云短信开通指引.md》仅作存档参考(若日后取得企业认证可再启用)。

---

## 1. 后端需要的配置项

把下面 6 个值填进 `server/.env`(本地)或 `deploy/.env`(服务器部署),两者都已被 `.gitignore` 忽略:

| 变量 | 含义 | 示例 |
|------|------|------|
| `SMTP_HOST` | SMTP 服务器地址 | `smtp.qq.com` |
| `SMTP_PORT` | 端口,默认 `465` | `465`(SSL)/ `587`(STARTTLS) |
| `SMTP_SECURE` | 是否使用 SSL,默认 `true` | 端口 465 用 `true`;端口 587 用 `false` |
| `SMTP_USER` | 发信邮箱账号(完整邮箱) | `你的邮箱@qq.com` |
| `SMTP_PASS` | **授权码 / 应用专用密码**(不是邮箱登录密码!) | 一串字母数字 |
| `SMTP_FROM` | 发件人,需与 `SMTP_USER` **同一邮箱** | `小善学习站 <你的邮箱@qq.com>` |

另有一个开关 `EMAIL_DEV_MODE`:

- `EMAIL_DEV_MODE=true` —— 演示模式,验证码直接显示在屏幕 / 日志上,不真正发邮件(本地调试用)。
- `EMAIL_DEV_MODE=false` —— 生产模式,走真实 SMTP 发信(上线必须设为 false)。

> **重要:** `SMTP_PASS` 几乎所有邮箱都要求填「授权码 / 应用专用密码」,**不是**你平时登录邮箱用的密码。
> 直接填登录密码通常会发信失败(报鉴权错误)。

## 2. 个人可用的免费 / 低成本选项

下面三种都适合个人,任选其一即可。

### 2.1 QQ 邮箱(推荐,国内最易上手)

- `SMTP_HOST=smtp.qq.com`
- `SMTP_PORT=465`
- `SMTP_SECURE=true`
- `SMTP_USER=你的QQ号@qq.com`
- `SMTP_PASS=`(授权码,见下)
- `SMTP_FROM=小善学习站 <你的QQ号@qq.com>`

获取授权码步骤:
1. 登录网页版 QQ 邮箱 → 「设置」→「账户」。
2. 找到「POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV 服务」一栏,**开启「IMAP/SMTP 服务」**。
3. 按提示用手机短信验证后,系统会生成一串**授权码**。
4. 把这串授权码作为 `SMTP_PASS` 填入(不是你的 QQ 密码)。

### 2.2 163 邮箱(网易)

- `SMTP_HOST=smtp.163.com`
- `SMTP_PORT=465`
- `SMTP_SECURE=true`
- `SMTP_USER=你的邮箱@163.com`
- `SMTP_PASS=`(授权码,见下)
- `SMTP_FROM=小善学习站 <你的邮箱@163.com>`

获取授权码步骤:
1. 登录网页版 163 邮箱 → 「设置」→「POP3/SMTP/IMAP」。
2. **开启「IMAP/SMTP 服务」**(部分账号需先开启「客户端授权密码」)。
3. 按提示验证后生成**授权码**,作为 `SMTP_PASS` 填入。

### 2.3 Gmail(海外 / 已开启两步验证的账号)

- `SMTP_HOST=smtp.gmail.com`
- `SMTP_PORT=587`(STARTTLS)或 `465`(SSL)
- `SMTP_SECURE=false`(用 587 时)/ `true`(用 465 时)
- `SMTP_USER=你的邮箱@gmail.com`
- `SMTP_PASS=`(应用专用密码,见下)
- `SMTP_FROM=小善学习站 <你的邮箱@gmail.com>`

获取应用专用密码步骤:
1. 先为 Google 账号**开启两步验证(2-Step Verification)**——这是生成应用专用密码的前提。
2. 进入 Google 账号 →「安全性」→「应用专用密码(App passwords)」。
3. 生成一个 16 位的**应用专用密码**,作为 `SMTP_PASS` 填入(不是你的 Google 登录密码)。

---

## 3. SMTP_FROM 的注意事项

- `SMTP_FROM` 的邮箱地址**必须与 `SMTP_USER` 一致**。个人邮箱通常要求发件人与认证账号相同,
  否则会被服务器拒绝(防伪造)。
- 格式:`显示名 <邮箱地址>`,例如:

  ```
  SMTP_FROM=小善学习站 <你的邮箱@qq.com>
  ```

  显示名是收件人看到的发件人名称;尖括号里必须是真实的发信邮箱。

---

## 4. 规模化提示(用户量变大时)

个人邮箱都有**每日发信上限**(几十到几百封不等),用户量上来后容易触发限额或被判垃圾邮件。
届时建议改用**事务邮件服务**,它们送达率更高、配额更大:

- Resend
- SendGrid
- 腾讯云邮件推送 / 阿里云邮件推送

好消息:本项目用的是**通用 SMTP(nodemailer)**,换服务商**只需改 `SMTP_*` 这几个值**
(host / port / secure / user / pass / from),后端代码无需改动。

---

## 5. 安全提示

- **授权码 / 应用专用密码等同于密码**:谁拿到都能以你的名义发信,务必妥善保管。
- 只放在 `server/.env`(本地)或 `deploy/.env`(部署),这两个文件**均已被 `.gitignore` 忽略**。
- **切勿**把授权码提交到 git、写进代码、或在聊天 / 截图中外泄。
- 一旦怀疑泄露,立即到对应邮箱的安全设置里**关闭并重新生成**授权码。
