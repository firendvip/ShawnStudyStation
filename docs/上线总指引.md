# 小善学习站 上线总指引（s.look3.cn）

## 现状说明

- look3.cn 目前是部署在腾讯云 CDN + COS 上的静态网站，已经完成 ICP 备案，并且已有 SSL 证书。
- s.look3.cn 是为这个新应用（小善学习站）单独准备的子域名。
- 应用代码已经放在私有 GitHub 仓库：firendvip/xiaoshan-study-station。
- 本次目标：把这个新应用部署到一台「腾讯云轻量应用服务器」上，并通过 https://s.look3.cn 访问。

## 怎么用这份指引

你将按顺序完成 5 个阶段。每个阶段末尾都有一个「粘贴块」（用四个反引号包起来的一整段文字）。请把整段「粘贴块」完整复制，粘贴给网页端的 Claude 浏览器插件（网页端 Claude 插件），让插件帮你在浏览器 / 腾讯云网页终端里一步步操作。

**一次只做一个阶段，做完一个再做下一个。** 不要几个阶段一起做。

## 准备好这些信息

- **服务器公网 IP**：在阶段 1 购买服务器后会拿到，请记下来，后面阶段都会用到。
- **QQ 邮箱授权码**：用于让应用通过 QQ 邮箱发送注册验证码邮件。请提前在 QQ 邮箱里开通 SMTP 服务并获取授权码。
- 其余的密码 / 密钥（数据库密码、JWT 密钥等）可以在部署现场临时生成，不用提前准备。

## 5 阶段一览

1. 购买轻量应用服务器
2. 开放防火墙端口
3. 给 s.look3.cn 添加 DNS 解析
4. 申请免费 SSL 证书（s.look3.cn）
5. 部署应用（OrcaTerm 网页终端）

---

## 阶段 1 — 购买轻量应用服务器

简介：

- 在腾讯云轻量应用服务器控制台买一台服务器，作为运行新应用的主机。
- 重点是地域要选国内（和域名备案一致）、配置别太小（建议 2核4G）、记下公网 IP。

````
你好 Claude 浏览器插件，请帮我完成「阶段 1：购买轻量应用服务器」。

请打开腾讯云轻量应用服务器控制台：https://console.cloud.tencent.com/lighthouse
然后一步步引导我购买一台轻量应用服务器，要点如下：

1. 地域：选择离用户近的国内地区（必须是国内地域，并且要与 look3.cn 域名备案所在的主体/地域一致）。不要选香港或海外地域。
2. 镜像：选择「应用镜像 Docker」；如果找不到，就选「系统镜像 Ubuntu 22.04」（如果选 Ubuntu，后面阶段 5 会再安装 Docker）。
3. 实例配置：建议选「2核4G」。请提醒我：2核2G 在服务器上构建 Next.js 镜像时可能内存不足，4G 更稳妥。
4. 系统盘：容量不少于 50G。
5. 设置实例的登录密码或密钥，并让我妥善保存。
6. 下单付款后，进入实例详情页，帮我找到并记下这台服务器的「公网 IP」（后面每个阶段都要用）。

费用说明：轻量 2核4G 国内大约每月数十元人民币，具体以页面实际显示为准，请以结算页金额为准。

请一步一步带我操作，每一步先告诉我点哪里、填什么，确认后再进行下一步。
````

## 阶段 2 — 开放防火墙端口

简介：

- 服务器默认会拦截外部访问，需要在防火墙里放行网站要用的端口。
- 要放行 22（SSH 登录）、80（HTTP）、443（HTTPS）三个端口。

````
你好 Claude 浏览器插件，请帮我完成「阶段 2：开放防火墙端口」。

请打开腾讯云轻量应用服务器控制台：https://console.cloud.tencent.com/lighthouse
进入我在阶段 1 购买的那台实例，找到「防火墙」面板，然后帮我放行以下 TCP 端口：

- 22（SSH，用于登录服务器）
- 80（HTTP）
- 443（HTTPS）

请逐条添加放行规则（协议选 TCP，来源可用默认全部）。添加完成后，请帮我再次确认这三个端口都已在放行列表里。

另外，请帮我再次记下这台服务器的「公网 IP」，下个阶段做 DNS 解析时要用。

请一步一步带我操作，每一步先告诉我点哪里、填什么。
````

## 阶段 3 — 给 s.look3.cn 添加 DNS 解析

简介：

- 让 s.look3.cn 这个子域名直接指向新服务器的公网 IP。
- 注意：这条解析是直连服务器、不走 CDN（和 look3.cn 主站不同）。

````
你好 Claude 浏览器插件，请帮我完成「阶段 3：给 s.look3.cn 添加 DNS 解析」。

请打开 DNSPod 控制台：https://console.dnspod.cn
（也可以在腾讯云「DNS 解析 DNSPod」里找到 look3.cn 的解析记录列表）

进入 look3.cn 的解析记录页面后，帮我新增一条记录：

- 主机记录：s
- 记录类型：A
- 记录值：填入我在阶段 1 拿到的服务器公网 IP
- 线路（解析线路）：默认
- TTL：默认

重点：这条记录是让 s.look3.cn 直接指向这台新服务器，不走 CDN（请不要把它配置成 CNAME 指向 CDN）。

保存后，解析生效（通常几分钟内），s.look3.cn 就会指向新服务器。请告诉我：可以在电脑命令行运行 ping s.look3.cn 来验证返回的 IP 是否就是阶段 1 的服务器公网 IP。

请一步一步带我操作，每一步先告诉我填什么。
````

## 阶段 4 — 申请免费 SSL 证书（s.look3.cn）

简介：

- 给 s.look3.cn 申请一张免费证书，这样网站才能用 https 安全访问。
- 用 DNS 验证方式，签发后下载 Nginx 格式的证书文件备用。

````
你好 Claude 浏览器插件，请帮我完成「阶段 4：申请免费 SSL 证书（s.look3.cn）」。

请打开腾讯云 SSL 证书控制台：https://console.cloud.tencent.com/ssl
然后帮我申请一张「免费 DV 证书」，要点如下：

1. 申请免费证书，绑定域名填写：s.look3.cn
2. 验证方式：选择「DNS 验证」。按提示在 DNSPod 里添加一条 TXT 记录用于验证；腾讯云通常支持「一键添加」到 DNSPod，请帮我点这个一键添加。
3. 等待证书签发（DNS 验证通过后会自动签发）。
4. 签发完成后，点「下载」，下载格式选择「Nginx」。会得到一个压缩包，里面通常有一个 .crt（或 fullchain）文件和一个 .key 文件。

请提醒我：稍后在阶段 5 部署时，要把这两个文件分别改名为 fullchain.pem 和 privkey.pem，放到服务器的 deploy/certs/ 目录里。请让我先把这个压缩包保存好。

请一步一步带我操作。
````

## 阶段 5 — 部署应用（OrcaTerm 网页终端）

简介：

- 这一步在服务器的网页终端（OrcaTerm）里运行命令，把应用真正跑起来。
- nginx 的 server_name 已经设好为 s.look3.cn，不需要你手动改。
- 按编号步骤依次执行即可。

````
你好 Claude 浏览器插件，请帮我完成「阶段 5：部署应用（OrcaTerm 网页终端）」。

开始前请先向我索取这两项信息：
- 服务器公网 IP
- QQ 邮箱授权码
其余值（POSTGRES_PASSWORD 数据库密码、JWT_SECRET 密钥）可以在现场用命令生成，不用我提供。

安全提醒：.env 文件和证书文件不要提交到 git（仓库的 .gitignore 已经忽略了它们），请不要把它们 push 上去。

首先：请打开腾讯云轻量应用服务器控制台 https://console.cloud.tencent.com/lighthouse ，进入我的实例，点击「登录」按钮，用 OrcaTerm 网页终端登录到服务器。登录成功后，按下面的编号步骤逐条执行命令（每执行一条，先告诉我预期结果，再继续下一条）：

1. 安装 Docker + compose 插件：运行 curl -fsSL https://get.docker.com | sh ，然后运行 apt-get install -y docker-compose-plugin ；最后验证 docker --version 和 docker compose version 都能正常输出版本号。（如果阶段 1 选的是「应用镜像 Docker」，可跳过安装，直接做这两条验证）

2. 生成只读部署密钥并加到 GitHub：运行 ssh-keygen -t ed25519 -C "look3-vps-deploy" -f ~/.ssh/id_ed25519 -N "" ，然后运行 cat ~/.ssh/id_ed25519.pub 复制输出的公钥；打开 https://github.com/firendvip/xiaoshan-study-station/settings/keys ，点「Add deploy key」，把公钥粘贴进去，**不要勾选 Allow write access**，保存；再回到终端运行 ssh -T git@github.com （首次会提示是否信任，输入 yes 回车）。

3. 克隆代码：运行 git clone git@github.com:firendvip/xiaoshan-study-station.git && cd xiaoshan-study-station/deploy

4. 配置环境变量：运行 cp .env.example .env && nano .env ，在文件里填写以下内容：
   - DOMAIN=s.look3.cn
   - POSTGRES_PASSWORD=（一个强随机密码）
   - JWT_SECRET=$(openssl rand -hex 32) （也可以先单独运行 openssl rand -hex 32 生成一串，再把结果粘贴进去）
   - EMAIL_DEV_MODE=false
   - SMTP_HOST=smtp.qq.com
   - SMTP_PORT=465
   - SMTP_SECURE=true
   - SMTP_USER=1074650916@qq.com
   - SMTP_PASS=（你的 QQ 邮箱授权码）
   - SMTP_FROM=小善学习站 <1074650916@qq.com>
   填好后在 nano 里按 Ctrl+O 回车保存，再按 Ctrl+X 退出。

5. 上传证书：把阶段 4 下载的两个文件分别改名为 fullchain.pem 和 privkey.pem，放进 deploy/certs/ 目录。可以用 nano 新建文件后把内容粘贴进去（例如 nano certs/fullchain.pem、nano certs/privkey.pem），或者使用 Lighthouse 控制台的文件上传功能上传。

6. 启动：运行 docker compose up -d --build （首次构建比较久，请耐心等待）。完成后运行 docker compose ps ，确认 postgres、backend、pinpin、web 这四个服务的状态都是 Up。

7. 验证：运行 curl https://s.look3.cn/api/health ，期望返回 {"ok":true} ；然后用浏览器打开 https://s.look3.cn ，测试 注册（会收到真实邮件验证码）/ 登录 / 错字练习 / 自然拼读 这几个功能。

8. 排错：可以分别运行 docker compose logs -f backend 、docker compose logs -f pinpin 、docker compose logs -f web 查看日志。如果构建时提示内存不足，可以加 swap：依次运行 fallocate -l 2G /swapfile && chmod 600 /swapfile && mkswap /swapfile && swapon /swapfile ，然后重新运行 docker compose up -d --build 。

请一步一步带我操作，每一步先说明这条命令是做什么的、预期看到什么，再继续下一步。
````
