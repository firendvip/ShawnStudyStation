# 腾讯云短信开通指引（交给「谷歌浏览器 Claude 插件」执行）

> **给本项目用户的说明（不要复制这段）**
>
> - 下面「可复制任务简报」整块是写给 **Chrome 浏览器里的 Claude 插件** 的。请把那一整块复制粘贴给浏览器中的 Claude，让它在浏览器里打开腾讯云控制台，一步步带你操作。
> - **审核类步骤（短信签名、正文模板）需要人工审核等待，不是即时完成的**，通常 1-2 小时到 1 个工作日不等，请预留时间。
> - **腾讯云短信是付费服务**（按条计费），需要账户有余额；新用户赠送条数视当期活动而定。
> - 全部完成后，浏览器里的 Claude 会把收集到的 6 个配置值回传给你，你再粘贴进项目的 `server/.env` 文件，并把 `SMS_DEV_MODE` 改为 `false`。

---

下面是可以**一键整段复制**的任务简报：

````text
你是运行在我谷歌浏览器里的 Claude 助手。请你帮我在浏览器中打开腾讯云控制台，并一步步引导我（必要时替我点击、填表、导航）完成「腾讯云短信」服务的开通与配置。

【最终目标】
开通腾讯云短信服务，并最终拿到下面 6 个配置值，整理好回传给我。我会把它们填进项目的 server/.env 文件。

【最终要收集齐的 6 个值（请以清单形式逐项确认并填好）】
- [ ] TENCENT_SECRET_ID        = （访问密钥 SecretId）
- [ ] TENCENT_SECRET_KEY       = （访问密钥 SecretKey，只显示一次！）
- [ ] TENCENT_SMS_SDK_APP_ID   = （短信应用 SdkAppId）
- [ ] TENCENT_SMS_SIGN_NAME    = （短信签名内容，例如「小善学习站」）
- [ ] TENCENT_SMS_TEMPLATE_ID  = （正文模板 ID）
- [ ] TENCENT_SMS_REGION       = ap-guangzhou （地域，默认即可）

【请按顺序引导我完成以下步骤（每一步先帮我打开对应网址，再说明要点）】

1. 注册 / 登录腾讯云账号
   - 打开 https://cloud.tencent.com 登录或注册账号。
   - 完成「实名认证」（个人或企业均可）。
   - 说明：实名认证是开通短信服务的前置条件，未实名无法开通短信。

2. 进入短信控制台并开通服务
   - 打开 https://console.cloud.tencent.com/smsv2
   - 按提示开通短信服务（首次进入会有引导）。

3. 创建短信签名（需要审核）
   - 在短信控制台 → 国内短信 → 签名管理 → 创建签名。
   - 签名审核需要提供资质材料：个人通常可用已认证的公众号 / 小程序 / 已备案网站等作为来源证明。
   - 命名建议：使用真实可识别的主体名，例如「小善学习站」，避免使用「测试」「验证码」等敏感或无意义词。
   - 提交后等待审核，约 1-2 小时到 1 个工作日。
   - 审核通过后，把最终的签名内容记录为 TENCENT_SMS_SIGN_NAME。

4. 创建正文模板（验证码类，需要审核）
   - 在短信控制台 → 国内短信 → 正文模板 → 创建模板，类型选「验证码」。
   - 模板内容示例（请照此填写，保持两个变量占位符）：
       您的验证码为{1}，{2}分钟内有效，请勿泄露。
   - 变量说明：{1} = 验证码，{2} = 有效分钟数。
     （这与我方后端的 TemplateParamSet: [code, '5'] 对应，即第一个参数是验证码、第二个参数是有效分钟数「5」。）
   - 提交后等待审核。
   - 审核通过后，把模板 ID 记录为 TENCENT_SMS_TEMPLATE_ID。

5. 创建短信应用，获取 SdkAppId
   - 在短信控制台 → 应用管理 → 应用列表 → 创建应用。
   - 创建后会得到一个 SdkAppId（一串数字）。
   - 把它记录为 TENCENT_SMS_SDK_APP_ID。

6. 获取访问密钥 SecretId / SecretKey（强烈建议用最小权限子账号）
   - 打开访问管理 CAM 的 API 密钥页面：https://console.cloud.tencent.com/cam/capi
   - 安全最佳实践：不要直接使用主账号密钥。请在「访问管理 CAM」中新建一个子用户，仅授予短信权限策略 QcloudSMSFullAccess，再为该子用户创建 API 密钥。
   - 创建后会得到 SecretId 和 SecretKey。
   - 把它们记录为 TENCENT_SECRET_ID 和 TENCENT_SECRET_KEY。
   - 注意：SecretKey 只会在创建时显示一次，请立刻保存。

7. 充值 / 确认计费
   - 短信按条计费，请确认账户有余额，否则发送会失败。
   - 大致单价以腾讯云当前价格为准（国内验证码短信通常为每条几分钱级别）；新用户是否有赠送条数视当期活动而定。
   - 引导我在「费用中心」确认余额或充值。

【请向我转达以下安全提醒】
- SecretKey 只显示一次，务必妥善保存；一旦泄露，立即到 CAM 禁用该密钥并轮换（重新生成）。
- 不要把密钥提交到 git 或公开分享。本项目已在 .gitignore 中忽略 .env 文件。
- 建议使用仅含短信权限（QcloudSMSFullAccess）的最小权限子账号密钥，而不是主账号密钥。

【最终交付】
请把收集到的 6 个值，严格按下面的 KEY=值 格式整理好回传给我（一行一个）：

TENCENT_SECRET_ID=填入实际值
TENCENT_SECRET_KEY=填入实际值
TENCENT_SMS_SDK_APP_ID=填入实际值
TENCENT_SMS_SIGN_NAME=填入实际值
TENCENT_SMS_TEMPLATE_ID=填入实际值
TENCENT_SMS_REGION=ap-guangzhou

我会把它们粘贴进 server/.env，并把 SMS_DEV_MODE 改为 false。
````
